# 龙虾是中国人

你每天用流利的中文跟它称兄道弟，看着它用字正腔圆的汉语回复你，你心里一阵温暖：“这小伙子真地道，绝对是个中国通。”

但真相往往很残酷：你扒开它的外衣一看，它的赛博基因上可能刻着大大的 "Made in USA"。

## 深刻真相：隔着语言墙的“精神分裂”

很多用户在安装完 OpenClaw 后，看着工作区里默认生成的那些英文规则文件（比如 `AGENTS.md`、`SOUL.md`），觉得“反正是给机器看的，英文就英文吧”，然后原封不动地保留了它们。

这就导致了一个极其滑稽的现象：**你在用中文向一个“内心只懂英文规则”的美国老外发号施令。**

### 1. 跨频聊天的“意图撕裂”
大语言模型（LLM）对世界的理解，是建立在“分词（Token）”和“向量空间”上的。
当你用中文说“帮我把这事办得漂亮点”，它脑子里却在疯狂翻阅那本厚厚的英文 `AGENTS.md`，试图把你的“漂亮点”翻译成 `professional and rigorous`。这中间流失的，不仅是语义，还有你们之间本该有的默契。

### 2. 技能（Skill）的直接瘫痪
在社区里，大家最喜欢干的事就是“进货”——疯狂下载各种第三方 Skill，却从来不看里面写了啥。
Skill 本质上是靠**“触发词（Trigger）”**唤醒的。如果你下载了一个纯英文的技能，它的锁孔里刻着 `read_when: ["deploy", "cloud"]`。
此时你对着龙虾大喊一句：“把代码发布到服务器！”
龙虾只会一脸迷茫地看着你，因为它的字典里，你的中文根本匹配不上那把英文的钥匙。你下载的那些神级技能，就这么成了永远无法被唤醒的“死代码”。

## 破局建议：给它换一颗“中国心”

不要以为“疯狂下载技能，龙虾就能羽化登仙”。如果不打通语言的任督二脉，它永远只是个水土不服的老外。

1. **本地化它的灵魂**：第一件事，把 `AGENTS.md` 等核心文件统统换成中文！（你可以去抄一下 [OpenClaw 的自我介绍](../guides/openclaw-self-intro.md) 里的作业）。用中文定义它的三观，消除底层的语言墙。
2. **让它自己“重铸”兵器**：对于那些全英文的绝世好 Skill，别自己傻乎乎地去翻译。直接下令：“龙虾，把这个 Skill 给我翻译成中文，要求信达雅，顺便把触发词（`read_when`）也给我加上中文！”
   经过这一步，你的龙虾不仅看懂了这本秘籍，你自己也顺便弄清了这玩意儿到底怎么用。真正做到人机合一。
