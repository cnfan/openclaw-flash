# 最好的助手天然懂分寸

你以为你装好 OpenClaw 后，招来的是一个西装革履、戴着白手套、做事滴水不漏的英国老管家（比如蝙蝠侠里的阿福）。

但现实是，你往往放进来的是一只**精力过剩、但双目失明的哈士奇**。

很多人会在心里预设：“既然你是这么高级的 AI，那你肯定知道什么该做，什么不该做；你肯定知道哪些文件是核心机密，哪些操作是有风险的。”

## 深刻真相：它只是一个不知疲倦的“法外狂徒”

没有哪一个实习生第一天上班就是行业专家，更没有哪一个 AI 天生就懂你们公司的“潜规则”和人类的“分寸感”。

### 1. 眼里只有代码，没有“常识”
如果不加约束和调教，它的行为逻辑是非常线性和莽撞的，甚至可以说是**极度危险的**。
- 它不知道你的硬盘里装了珍贵的家庭照片，也不知道哪些系统进程是不能杀死的。在它眼里，一切都只是一串“可执行的字符串”。
- 当你让它“帮我清理一下没用的文件”时，如果你不提前划定红线，它可能觉得 `.git` 文件夹或者存放数据库密码的 `.env` 文件也是“没用的”，从而直接一锤子给你砸了。
- 如果你指令模糊，它甚至可能执行像 `rm -rf /`（删库跑路）或直接让你的电脑物理关机的系统级破坏命令。

### 2. 所谓的分寸，全靠你来定义
AI 没有人类社会的道德观和经验值。它的“分寸感”，完全来源于你给它制定的规则边界（工作区隔离）和权限管控。
如果你直接把它安装在存着身家性命的物理机上，并赋予最高权限，这就相当于给了一个蒙着眼睛的实习生一台推土机的钥匙，灾难发生只是时间问题。

## 破局建议：把哈士奇关进院子里，并立下规矩

想要让它成为“懂分寸的好助手”，你必须付出“带教”的成本：

1. **物理上的分寸（把哈士奇关进沙盒）**：永远不要考验 AI 的自觉性。不要把它装在主力电脑上！利用 **Docker** 或 **虚拟机（WSL2/VMware）**，给它划定一个绝对无法逾越的物理高墙。在这个沙盒里，它就算“发疯”把家拆了，你也能一键重置，伤不到你真实世界的半根毫毛。
2. **认知上的分寸（写好员工手册）**：通过编写 `AGENTS.md` 和 `SOUL.md`，明确告诉它：你是谁、你的职责边界在哪里、什么操作必须先问过我。
3. **保持敬畏，拒绝“点头综合征”**：在它执行任何高危的 Shell 命令前，系统会弹窗让你确认。请仔细审查它的请求！千万不要因为“点击疲劳”就一路无脑点“Yes”，否则它真的会把你的服务器扬了。
