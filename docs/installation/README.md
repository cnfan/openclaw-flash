# 安装Openclaw

## 优先沙盒安装

- 如果你的机器上有工作资料、隐私数据、密钥文件，优先使用 **沙盒安装**（WSL2 / VMware / Docker）。
- 所谓沙盒，你可以理解成你的主机是上帝，你能操控它，但是它却不了解你，开放是单向的，风险是封闭的，可控的
- 如果未来 OpenClaw 被攻击，沙盒可以缩小攻击面，降低主系统和敏感数据暴露风险。
- 另外沙盒还能最大化最容易做安全防护，市面上的一切安全类产品都强调沙盒原则

## 为什么我推荐虚拟机 / WSL2 / Docker

核心原因都是安全边界：

- **隔离性**：把 OpenClaw 放到独立环境，和你的主系统分层。
- **可回滚**：虚拟机有快照，Docker 有镜像与容器重建能力。
- **更可控**：环境、权限、网络策略更容易单独收敛。

从安全性角度，优先级建议：

1. Docker（Linux / macOS）
2. WSL2（Windows）
3. VMware（Windows，可选）
4. 直接安装（最后选项）

## 为什么推荐原版，不推荐“厂商套壳版 / 魔改版”

OpenClaw 迭代节奏非常快，一日多更，我看到一天更新十多次，像一辆持续升级的赛车。对普通用户来说，越接近原版，越不容易踩版本坑。

不推荐套壳版的主要原因：

- **更新滞后风险**：上游一旦快迭代，二次封装版往往跟不上。
- **兼容性风险**：大改后容易偏离官方行为，问题定位更难。
- **运维绑定风险**：有些方案把部署、算力、能力包强绑定到平台侧。
- **技能资产锁定风险**：如果 skill 只能云端托管、无法本地学习和迁移，长期成本与可控性都会变差。

目前的套壳版，除了网易的开源了，其余的我看都是有很多小心思的，以至于我后来也不看了。这些小心思包括但不限于

- **想当llm中间商，趁机做中国版的openRouter**
- **想做闭源skills，未来付费使用**
- **想做入口，做下一个百度或者腾讯**

建议优先使用原版流程，保持可迁移、可学习、可审计

### Linux
linux 是我最推荐的方式，对于很多家庭用户或者企业用户，24小时开机的nas非常适合，我自己就是在
自己的nas上安装的。但是nas安装注意一个事情，就是后续使用不要使用官方的浏览器shell安装飞书等
插件，会有一些问题。需要使用windows或者mac的原生终端ssh登录到docker镜像实例内部来安装。
虽然我不推荐直接安装，但是如果你是windows沙盒或者一个专门用来跑Openclaw的主机，我还是推荐的，
只要你把以后的操作接口最小化到飞书、微信、QQ，那么就是安全的。

- 首选： [Linux 安装方法 1：Docker 安装（推荐）](./refrences/linux-docker.md)
- 直接安装（APT）： [Linux 安装方法 2：直接安装 (APT)](./refrences/linux-native-apt.md)
- 直接安装（YUM）： [Linux 安装方法 3：直接安装 (YUM)](./refrences/linux-native-yum.md)

### mac OS
一般买mac mini的专门用来安装Openclaw的都是老板，但是我想说下我的观点，我不建议在没有使用
场景的情况下买mac mini来安装，对于有需要的老板，我还是祝福的，我自己的mac 用了十年还能流畅
跑win11和mac OS，目前Apple还在维护 ipad air2，这些理念真的是和胖东来一样，真心换真品。

- 首选： [macOS 安装方法 1：Docker 安装（推荐）](./refrences/macos-docker.md)

## 选型建议

- 你是个人 Windows 用户：先走 WSL2。
- 你希望“可回滚 + 全隔离”：走 VMware。
- 你是 Linux/macOS 用户：优先 Docker。
- 你只做临时测试机且无敏感数据：才考虑直接安装。
