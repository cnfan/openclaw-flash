# 专栏作家的“影分身”工作流

对于文字工作者（作家、自媒体人、编剧）来说，最大的痛点不是“写不出字”，而是“找资料、理大纲、排版排到头秃”。

如果你还在把龙虾当成一个“帮我写一段话”的续写工具，那你就完全浪费了它的工程化能力。下面来看一个高阶的文字工作者是如何利用龙虾搭建“自动化爆款流水线”的。

说明：以下流程与数值指标为设想示例，用于说明如何拆分角色与编排步骤；实际可行性与收益取决于数据来源权限、平台规则、内容质量、以及你是否具备相应的脚本/接口能力。

## 痛点：内容产出链路太重

一个专栏作家要产出一篇高质量的深度长文，通常需要经历：
1. 去外网和各大研报网站疯狂爬取最近 3 天的行业资讯。
2. 翻译、总结、提炼核心观点。
3. 根据自己的文风列大纲。
4. 撰写初稿，并配上数据图表。
5. 针对微信公众号、小红书、知乎进行不同排版格式的转换。

这套流程走下来，写字只占 20% 的时间，剩下 80% 全在干体力活。

## 高阶解法：构建“编辑部”式多智能体协同（示例）

高级玩家不会让一只龙虾干完所有事，而是通过 OpenClaw 的 **Sub-Agents（子智能体）** 架构，在本地拉起了一个“虚拟编辑部”。

### 1. 情报虾（资料收集员）
- **技能（Skill）配置**：例如挂载联网搜索、解析 PDF、翻译等能力。
- **定时任务（Cron）**：例如设定在固定时间抓取/汇总指定来源的信息，形成一份《晨间情报简报》，存入工作区的 `briefing.md`。

### 2. 主笔虾（核心创作者）
- **灵魂配置（SOUL.md）**：注入了作家本人的历史爆款文章作为语料，让它的文风充满“犀利、幽默、一针见血”的个人特色。
- **工作流**：作家早上起床喝咖啡时，只需对主笔虾说：“基于今天情报虾整理的 `briefing.md`，挑第二个热点，帮我列一个三段式的文章大纲。”确认大纲无误后，主笔虾自动完成初稿扩写。

### 3. 排版虾（运营助理）
- **技能（Skill）配置**：例如挂载格式转换、图片处理/生成等能力。
- **工作流**：初稿定稿后，排版虾接手，把内容转换为不同平台所需的格式，并按需准备配图素材。

## 结果：在条件满足时的可预期提效

通过 `Cron + Sub-Agents + 定制 Skills` 的组合，重复性环节（收集、整理、格式转换）可以更自动化；作者把主要精力留给选题判断、结构把控与关键段落的改写。
