# 如何成为高级玩家

当你不满足于仅仅和龙虾“聊天”，当你开始厌倦每天机械地给它发同样的指令时，你就站在了“高级玩家”的门槛上。

高级玩家不需要具备从零开发大型软件的能力，但**必须具备极强的工程化思维和工作流编排能力**。要成为高级玩家，你需要彻底玩明白以下三个核心机制。

## 1. 玩明白“定时任务（Cron）与隔离会话”

初级玩家每天早上手动打字：“龙虾，帮我总结一下今天的新闻。”
高级玩家的龙虾会在每天早晨 7 点，自动把整理好的精美早报推送到微信里。

成为高级玩家的第一步，就是让龙虾学会“自己动”。
- **精通 Cron 语法**：你必须学会使用 `openclaw cron add` 命令，灵活设置 `every 2h`（每两小时）或 `0 7 * * *`（每天早上 7 点）的后台任务。
- **死守“隔离原则”**：你必须深刻理解 `session: isolated`（隔离会话）的价值。所有定时生成的垃圾日志、冗长报表，统统扔进隔离会话，绝对不让它们污染你日常交流的主会话（`main`）。

## 2. 深度定制与汉化你的 Skills

初级玩家到处求人：“有没有好用的 Skill 推荐？”
高级玩家冷冷一笑：“没有趁手的兵器，我自己打一把。”

Skill（技能）是龙虾的手脚。成为高级玩家，你必须掌握如何揉捏这些手脚：
- **拒绝拿来主义**：不要下载了一个全英文的 Skill 就直接用。你要学会指挥龙虾，把这个 Skill 逐行翻译成中文，并加上中文的 `read_when` 触发词。
- **微调脚本逻辑**：就算你不会写代码，你也要学会用自然语言命令龙虾：“修改这个 `weather.js` 技能，以后查天气时，必须顺便附带穿衣建议。”
- **渐进式披露**：你必须学会管理工作区里的 Skill 目录，利用触发词做到按需加载，而不是一股脑全塞给大模型导致它“精神分裂”。

## 3. 拆分 Agent，打造“多智能体协同”团队

初级玩家试图打造一个“全知全能的超级龙虾”，把写代码、画图、做报表的规矩全塞进一个 `AGENTS.md` 里，结果龙虾每天都在胡言乱语。

高级玩家深知大模型“会分心”的物理限制。他们的绝招是：**拆分与协作（Sub-Agents）**。
- **主控大脑（包工头）**：设立一个主 Agent，它的唯一任务就是理解你的需求，然后把任务分发下去。它不干脏活累活。
- **专职子 Agent（打工仔）**：
  - 建立一个只懂前端代码、只挂载了编译 Skill 的 `前端虾`。
  - 建立一个只懂检索、只挂载了爬虫 Skill 的 `情报虾`。
- **协同工作流**：主 Agent 收到需求后，唤醒 `情报虾` 去查资料，拿到资料后丢给 `前端虾` 去写页面。各个子 Agent 在自己纯净的 Workspace 里干活，互不干扰，彻底消灭“幻觉”和“漂移”。

当你能熟练地指挥一个主 Agent 带着三个子 Agent 帮你跑通一个自动化项目时，恭喜你，你已经是一名真正的 OpenClaw 高级玩家了。
