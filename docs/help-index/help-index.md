## 二、后续建议补充

- 快速开始：从安装到第一次对话的最短路径
- 渠道与会话机制：私聊、群聊、线程、session 的对应关系
- pairing 配对机制：陌生用户怎么授权接入
- memory 使用指南：`MEMORY.md` 和 `memory/YYYY-MM-DD.md` 怎么分工
- sandbox 与安全：工作区不是硬隔离，怎么减少执行风险
- 故障排查：安装失败、渠道不收消息、模型不可用、日志怎么看
- 升级与备份：怎么升级 OpenClaw，怎么备份工作区和配置
- 团队规范：如何写 `AGENTS.md`、如何写技能、如何管理工作区仓库

## 六、国内访问建议

如果你主要在中国大陆使用，实际体验上通常要优先准备这几件事：

- 一个稳定可访问 GitHub 的网络环境
- 一个能稳定访问 npm 的镜像或代理策略
- 明确哪些页面是官方，哪些是社区镜像或导航站

原因很简单：

- 官方代码和文档更新最快
- 社区中文站更适合查教程
- 但真正的版本事实，应该优先以官方文档和 GitHub 仓库为准
